# Open Order Lite — 1-Day 워크샵 경량화 설계서

> 대상: SAP 파트너 1-Day 워크샵 수강생  
> 목표: ERP CBO → Clean Core DSP 전환 핵심 개념을 3+3시간 내에 완주

---

## 1. 경량화 원칙

| 항목 | Full Version | Lite Version (이 문서) |
|------|-------------|----------------------|
| ABAP 조회 테이블 | 10개 (VBAK~KNKK) | **3개** (VBAK, VBAP, KNA1) |
| Z-Table 필드 수 | 29개 | **13개** |
| 계산 로직 | Aging, 납품율, 청구율, 신호등, 신용초과 | **Aging 구간만** |
| RF 소스 수 | 5개 | **2개** (2LIS_11_VAHDR + 2LIS_11_VAITM) |
| TF 조인 수 | 5-way JOIN | **2-way JOIN** (헤더 + 아이템) |
| AM 지표 수 | BASE 8 + CALC 3 + RESTR 4 | **BASE 3 + RESTR 4** |

---

## 2. 전체 흐름

```
[ AS-IS: ERP CBO — 오전 세션 ]

ZSD_OO_LITE (ABAP Report)
 ├─ VBAK (오더 헤더)
 ├─ VBAP (오더 아이템)
 └─ KNA1 (고객 마스터)
        ↓ 경과일/Aging 계산
 ZSDT_OO_LITE (Z-Table 저장)

↓ ↓ ↓ 문제 제기: "이 Z-Table을 DSP 소스로 쓰면 Clean Core 위반!" ↓ ↓ ↓

[ TO-BE: SAP Datasphere — 오후 세션 ]

2LIS_11_VAHDR ──RF──→ LT_OO_VAHDR ─┐
2LIS_11_VAITM ──RF──→ LT_OO_VAITM ─┴──→ TF_OO_LITE ──→ TT_OO_LITE
                                          (JOIN + AGING)       │
                                                        V_OO_LITE_F (Fact View)
                                                                │
                                                  V_MONTH_COMPARISON (Helper)
                                                                │
                                                        AM_OO_LITE (Analytic Model)
```

### 3-2. 조회 로직 (3 STEP)

```
STEP 1: VBAK — 오더 헤더 (audat 기간, vkorg, vbtyp='C')
STEP 2: VBAP — 오더 아이템 (abgru=' ', matnr 조건)
STEP 3: KNA1 — 고객 마스터 (고객명 조회)
STEP 4: MERGE — 통합 + Aging 계산
```

### 3-3. Aging 계산 (핵심 로직)

```abap
elapsed_days = sy-datum - ls_vbak-audat.
CASE
  WHEN elapsed_days <= 30  → aging_grp = '030'
  WHEN elapsed_days <= 60  → aging_grp = '060'
  WHEN elapsed_days <= 90  → aging_grp = '090'
  ELSE                     → aging_grp = '90+'
ENDCASE.
```

## 4. TO-BE: DSP 오브젝트 상세

### 4-1. RF_OO_LITE (Replication Flow)

| 소스 DataSource | 타겟 Local Table | 설명 |
|----------------|-----------------|------|
| `2LIS_11_VAHDR` | `LT_OO_VAHDR` | 오더 헤더 (VKORG, KUNNR, AUDAT, NETWR 등) |
| `2LIS_11_VAITM` | `LT_OO_VAITM` | 오더 아이템 (MATNR, KWMENG, NETWR 등) |

- 소스 커넥션: `S4_HANA` (connectionType: ABAP)
- 컨테이너: `/ODP_BW`
- Load Type: `REPLICATE` (Delta)

### 4-2. TF_OO_LITE (Transformation Flow)

**조인:**
```sql
LT_OO_VAHDR  INNER JOIN  LT_OO_VAITM
  ON LT_OO_VAHDR.VBELN = LT_OO_VAITM.VBELN
```

**계산 컬럼 (ABAP → SQL):**

| 컬럼 | ABAP | TF SQL |
|------|------|--------|
| AUDAT_YM | `audat(6)` | `LEFT(AUDAT, 6)` |
| ELAPSED_DAYS | `sy-datum - audat` | `DAYS_BETWEEN(AUDAT, CURRENT_DATE)` |
| AGING_GRP | IF/ELSEIF | `CASE WHEN ELAPSED_DAYS <= 30 THEN '030' WHEN ELAPSED_DAYS <= 60 THEN '060' WHEN ELAPSED_DAYS <= 90 THEN '090' ELSE '90+' END` |

### 4-4. V_OO_LITE_F (Fact View)

- 소스: `TT_OO_LITE`
- 어노테이션: `ANALYTICAL_FACT`
- BASE 지표 선언 (`@Aggregation.default: SUM`):

| 컬럼 | 집계 | AM에서 역할 |
|------|------|-----------|
| ORD_QTY | SUM | 오더 수량 |
| ORD_AMT | SUM | 오더 금액 |
| ELAPSED_DAYS | SUM | (보조용) |

### 4-5. AM_OO_LITE (Analytic Model)

**차원 (Attributes):**
VKORG, SPART, KUNNR, MATNR, MATKL, AUART, WAERK, AUDAT, AUDAT_YM, AGING_GRP

**BASE 지표:**

| 기술명 | 레이블 |
|--------|--------|
| ORD_AMT | 오더 금액 |
| ORD_QTY | 오더 수량 |

**RESTRICTION 지표 (기간 비교 — 핵심 학습 포인트):**

| 기술명 | 레이블 | 조건 |
|--------|--------|------|

| PREV_MONTH_AMT | 전월 오더금액 | AUDAT_YM = 전월 |
| CURR_YTD_AMT | 당년누계 오더금액 | 당년1월 ≤ AUDAT_YM ≤ 당월 |
| PREV_YTD_AMT | 전년누계 오더금액 | 전년1월 ≤ AUDAT_YM ≤ 전년동월 |
