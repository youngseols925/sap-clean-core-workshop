## 5-3. JOIN 추가 — LT_OO_VAITM (아이템)

1. 캔버스 빈 곳에 **Join** 노드 추가
2. `LT_OO_VAHDR (H)` → Join 노드 연결
3. Join 노드에 **두 번째 Source** 추가: `LT_OO_VAITM` (별칭: `I`)
4. Join 조건 설정:

| 좌측 (H) | 연산자 | 우측 (I) |
|----------|--------|----------|
| `DOC_NUMBER` | `=` | `DOC_NUMBER` |

5. Join 유형: **`INNER JOIN`**

> 💬 **강사 설명**:  
> "ABAP에서 `LOOP AT gt_vbak ... LOOP AT gt_vbap WHERE vbeln = ...` 이중 루프로 하던 것을  
> 여기서는 INNER JOIN 한 줄로 처리합니다."

```
[📸 스크린샷: INNER JOIN 설정 — H.DOC_NUMBER = I.DOC_NUMBER]
```

## 5-4. JOIN 추가 — LT_OO_CUST (고객 마스터)

1. 첫 번째 Join 노드에서 **또 다른 Join** 노드 추가
2. 두 번째 Source: `LT_OO_CUST` (별칭: `C`)
3. Join 조건:

| 좌측 | 연산자 | 우측 (C) |
|------|--------|----------|
| `H.SOLD_TO` | `=` | `CUSTOMER` |

4. Join 유형: **`LEFT JOIN`**

> 💬 **왜 LEFT JOIN인가?**  
> 고객명이 없어도 오더 데이터를 잃으면 안 되기 때문입니다.  
> `INNER JOIN`이면 `LT_OO_CUST`에 없는 고객의 오더가 통째로 사라집니다.

```
[📸 스크린샷: LEFT JOIN 설정 — H.SOLD_TO = C.CUSTOMER]
```

## 5-5. Projection (출력 컬럼) 설정

Join 이후 **Projection** 노드 추가 → 아래 컬럼을 추가합니다:

### 직접 매핑 컬럼 (그대로 가져오는 것)

| 출력 컬럼명 | 소스 | 소스 컬럼 |
|------------|------|---------|
| `VBELN` | H | `DOC_NUMBER` |
| `POSNR` | I | `S_ORD_ITEM` |
| `AUART` | H | `DOC_TYPE` |
| `AUDAT` | H | `AUDAT` |
| `VKORG` | H | `SALES_ORG` |
| `VTWEG` | H | `DISTR_CHAN` |
| `SPART` | H | `DIVISION` |
| `KUNNR` | H | `SOLD_TO` |
| `KUNNR_NAME` | C | `NAME` |
| `LAND1` | C | `COUNTRY` |
| `MATNR` | I | `MATERIAL` |
| `MATKL` | I | `MATL_GROUP` |
| `ARKTX` | I | `SHORT_TEXT` |
| `MEINS` | I | `BASE_UOM` |
| `VRKME` | I | `SALES_UOM` |
| `WAERK` | H | `CURRENCY` |
| `ORD_QTY` | I | `ORD_QTY` |
| `ORD_AMT` | I | `NET_VALUE` |

### 계산 컬럼 (Calculated Columns) — 핵심!

> **+ Add Calculated Column** 버튼으로 추가

**① AUDAT_YM — 오더 생성년월**

| 항목 | 값 |
|------|-----|
| 컬럼명 | `AUDAT_YM` |
| 타입 | `String (6)` |
| 수식 | `LEFT(TO_VARCHAR(H.AUDAT, 'YYYYMMDD'), 6)` |

> 💬 **ABAP 대응**: `ls_vbak-audat(6)` → HANA SQL `LEFT(TO_VARCHAR(AUDAT,'YYYYMMDD'),6)`

**② ELAPSED_DAYS — 경과일수**

| 항목 | 값 |
|------|-----|
| 컬럼명 | `ELAPSED_DAYS` |
| 타입 | `Integer` |
| 수식 | `DAYS_BETWEEN(H.AUDAT, CURRENT_DATE)` |

> 💬 **ABAP 대응**: `sy-datum - ls_vbak-audat` → HANA SQL `DAYS_BETWEEN(AUDAT, CURRENT_DATE)`  
> **중요**: ABAP은 배치 실행 시점이 기준, HANA는 **항상 오늘 날짜** 기준 → 항상 최신 Aging!

**③ AGING_GRP — Aging 구간**

| 항목 | 값 |
|------|-----|
| 컬럼명 | `AGING_GRP` |
| 타입 | `String (3)` |
| 수식 | *(아래 참조)* |

```sql
CASE
  WHEN DAYS_BETWEEN(H.AUDAT, CURRENT_DATE) <= 30 THEN '030'
  WHEN DAYS_BETWEEN(H.AUDAT, CURRENT_DATE) <= 60 THEN '060'
  WHEN DAYS_BETWEEN(H.AUDAT, CURRENT_DATE) <= 90 THEN '090'
  ELSE '90+'
END
```

> 💬 **ABAP 대응**: `IF elapsed_days <= 30 ... ELSEIF ... ELSE` → SQL `CASE WHEN`

```

```

## 5-7. Target 설정 — TT_OO_LITE

1. **Target** 노드 클릭
2. **Create New Table** 선택 → 이름: `TT_OO_LITE`
3. Load Type: **`Truncate and Insert`** 선택

> 💬 **Truncate and Insert란?**  
> TF 실행 시 기존 데이터를 전부 지우고 새로 채웁니다.  
> ABAP의 `DELETE FROM zsdt_oo_lite ... INSERT ... FROM TABLE` 패턴과 동일합니다.

```
[📸 스크린샷: Target = TT_OO_LITE, Load Type = Truncate and Insert]
```
