# AS-IS 세션 ① — Z-Table 생성 (SE11)
> **소요시간**: 60분  
> **목표**: ZSDT_OO_LITE 테이블을 SE11에서 직접 만들어본다

---

## 1-1. SE11 트랜잭션 열기

1. SAP GUI 실행 → ERP 로그인
2. 커맨드 박스에 **`SE11`** 입력 후 Enter

   ```
   [📸 스크린샷: SE11 시작 화면 — "데이터베이스 테이블" 라디오 버튼 선택된 상태]
   ```

3. **데이터베이스 테이블** 라디오 버튼 선택 (기본값)
4. 입력 필드에 **`ZSDT_OO_LITE`** 입력
5. **신규** 버튼 클릭 (또는 `F5`)

---

## 1-2. 테이블 기본 정보 입력

화면 상단 **속성** 탭에서 아래 내용 입력:

| 필드 | 입력값 |
|------|--------|
| 간략 설명 | `Open Order Lite (워크샵)` |
| 전달 클래스 | `A` |
| 데이터 브라우저/테이블뷰 유지관리 | `허용, 제한 없음` |

```
[📸 스크린샷: 속성 탭 — 간략 설명, 전달 클래스 입력 완료 상태]
```

---

## 1-3. 필드 탭 — 필드 입력

상단 **필드** 탭 클릭 후 아래 순서대로 필드를 입력합니다.

> 💡 **팁**: `MANDT` 필드는 직접 입력하지 않아도 됩니다.  
> 포함 구조 `MANDT` 또는 `CLIENT` 입력 시 자동 추가됩니다.

| 필드명 | 키 | 초기값 | 데이터 요소 | 설명 |
|--------|:--:|:------:|------------|------|
| `MANDT` | ✅ | ✅ | `MANDT` | 클라이언트 |
| `VBELN` | ✅ | ✅ | `VBELN_VA` | 판매 오더 번호 |
| `POSNR` | ✅ | ✅ | `POSNR_VA` | 오더 아이템 |
| `AUDAT` | | ✅ | `AUDAT` | 오더 생성일 |
| `AUDAT_YM` | | ✅ | `SPMON` | 오더 생성년월 |
| `VKORG` | | | `VKORG` | 영업 조직 |
| `SPART` | | | `SPART` | 제품군 |
| `KUNNR` | | | `KUNNR` | 고객 코드 |
| `MATNR` | | | `MATNR` | 자재 번호 |
| `MATKL` | | | `MATKL` | 자재 그룹 |
| `ORD_QTY` | | | `KWMENG` | 오더 수량 |
| `ORD_AMT` | | | `WERTV8` | 오더 금액 |
| `WAERK` | | | `WAERK` | 통화 키 |
| `ELAPSED_DAYS` | | | *(직접입력)* | 경과일수 |
| `AGING_GRP` | | | *(직접입력)* | Aging 구간 |

```
[📸 스크린샷: 필드 탭 — 전체 필드 입력 완료 상태]
```

> ⚠️ **주의**: `ELAPSED_DAYS`는 데이터 요소가 없습니다.  
> 데이터 요소 칸을 비우고 **직접 유형** 체크 → 유형: `INT4`, 길이: `10` 입력

> ⚠️ **주의**: `AGING_GRP`도 직접 유형 → 유형: `CHAR`, 길이: `3` 입력

---

## 1-4. Currency/Quantity Fields 탭 설정

`ORD_AMT` 필드는 금액(CURR) 타입이므로 통화 참조 필드를 지정해야 합니다.

1. 상단 **통화/수량 필드** 탭 클릭
2. `ORD_AMT` 행에서 **참조 필드**: `WAERK` 입력
3. **참조 테이블**: `ZSDT_OO_LITE` 입력

```
[📸 스크린샷: 통화/수량 필드 탭 — ORD_AMT → WAERK 참조 설정]
```

---

## 1-5. Technical Settings 설정

메뉴 **추가 정보 → Technical Settings** 클릭:

| 항목 | 값 |
|------|----|
| Data Class | `APPL0` |

| Buffering | `버퍼링 허용 안 함` |

**저장** 후 **뒤로** 버튼으로 필드 화면 복귀

```
[📸 스크린샷: Technical Settings 입력 완료]
```

---

## 1-6. 활성화

1. **활성화** 버튼 클릭 (또는 `Ctrl+F3`)
2. 활성화 로그에 오류 없으면 성공 ✅
3. SE16에서 **`ZSDT_OO_LITE`** 조회 → 빈 테이블 확인

```
[📸 스크린샷: 활성화 성공 메시지]
```

# AS-IS 세션 ② — ABAP Report 작성 (SE38)
> **소요시간**: 90분  
> **목표**: ZSD_OO_LITE 프로그램을 SE38에서 임포트하고 실행해본다

---

## 2-1. SE38 트랜잭션 열기

1. 커맨드 박스에 **`SE38`** 입력 후 Enter
2. 프로그램 이름에 **`ZSD_OO_LITE`** 입력
3. **신규** 버튼 클릭

```
[📸 스크린샷: SE38 시작 화면]
```

4. 속성 팝업:
   - 제목: `Open Order 현황 (Lite)`
   - 유형: `실행 가능 프로그램`
   - 상태: `테스트 프로그램`
   - 패키지: `$TMP` (임시 패키지)
5. **저장** 클릭

---

## 2-2. 소스코드 입력

ABAP 에디터가 열리면 **전체 선택(`Ctrl+A`)** 후 삭제 → 아래 코드를 붙여넣기:

> 📎 **코드 파일**: `lite/abap/programs/ZSD_OO_LITE.abap` (GitHub 레포 참조)

```abap
*&---------------------------------------------------------------------*
*& Report  ZSD_OO_LITE
*& Open Order 진행현황 조회 (워크샵 경량화 버전)
*& VBAK + VBAP 만 사용
*&---------------------------------------------------------------------*
REPORT zsd_oo_lite
  LINE-SIZE 255
  LINE-COUNT 65
  MESSAGE-ID zmsd.
```

> 💡 전체 코드는 강사가 제공하는 GitHub 링크에서 다운로드하세요.

---

## 2-3. 코드 구조 설명

코드를 붙여넣기 전, 강사가 전체 흐름을 설명합니다.

```
STEP 1: f_get_vbak  — VBAK에서 미완료 오더 헤더 가져오기
             ↓      (WHERE vbtyp='C' AND gbstk<>'C')
STEP 2: f_get_vbap  — VBAP에서 거부 안 된 아이템 가져오기
             ↓      (WHERE abgru=' ')
STEP 3: f_merge_data — VBAK × VBAP 매칭 + Aging 계산
             ↓
       [Aging 계산]
       elapsed_days = sy-datum - audat
       030 / 060 / 090 / 90+ 구간 분류
             ↓
STEP 4: f_save_ztable — ZSDT_OO_LITE에 저장 (p_save='X' 일때)
STEP 5: f_show_alv    — ALV Grid로 결과 출력
```

> 💬 **강사 설명 포인트**:  
> "10개가 넘는 ABAP 테이블에서 직접 SELECT 하는 복잡한 프로그램도 있지만,  
> 오늘은 핵심 개념 이해를 위해 VBAK + VBAP만 쓰는 Lite 버전으로 합니다."
