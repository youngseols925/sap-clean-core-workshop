# TO-BE 세션 ① — Replication Flow (RF_OO_LITE)
> **소요시간**: 60분  
> **목표**: ERP Standard DataSource → DSP Local Table 복제 파이프라인을 만든다

---

## 4-1. DSP 접속 및 Space 이동

1. 브라우저에서 DSP URL 접속:  
   `https://poc-dsp-1.ap12.hcs.cloud.sap`
2. 로그인 후 좌측 상단 **Space 선택** → `WORKSHOP` 선택

```
[📸 스크린샷: DSP 메인 화면 — WORKSHOP Space 선택]
```

---

## 4-2. RF_OO_LITE 신규 생성

1. 좌측 메뉴 **Data Integration** → **Replication Flows** 클릭
2. **New** 버튼 클릭
3. 이름 입력: `RF_OO_LITE`
4. **Create** 클릭

```
[📸 스크린샷: Replication Flow 신규 생성 화면]
```

---

## 4-3. Source Connection 설정

RF 편집 화면에서:

1. **Source** 섹션 → **Connection** 드롭다운 클릭
2. `S4_HANA` 선택 (connectionType: ABAP)
3. **Container** 드롭다운 → `/ODP_BW` 선택

```
[📸 스크린샷: Source Connection = S4_HANA, Container = /ODP_BW 선택 완료]
```

---

## 4-4. Task 1 추가 — 2LIS_11_VAHDR (오더 헤더)

1. **Add Task** 버튼 클릭
2. **Source Object** 검색창에 `2LIS_11_VAHDR` 입력 → 선택

```
[📸 스크린샷: Source Object 검색 — 2LIS_11_VAHDR 선택]
```

3. **Target Table** 이름 자동 생성됨 → **`LT_OO_VAHDR`** 로 변경
4. **Load Type**: `Initial and Delta` 선택

> 💬 **강사 설명**:  
> "`2LIS_11_VAHDR`가 VBAK 역할을 하는 Standard DataSource입니다.  
> `Initial` 은 전체 데이터 최초 로딩, `Delta`는 이후 변경분만 자동 복제됩니다."

---

## 4-5. Projection 설정 (필요 필드만 선택)

Task 1 클릭 → **Mapping** 탭:

아래 필드만 체크 (나머지는 체크 해제):

| Source 필드 | Target 컬럼명 | 설명 |
|------------|--------------|------|
| `DOC_NUMBER` | `DOC_NUMBER` | 판매오더 번호 (Key) |
| `AUDAT` | `AUDAT` | 오더 일자 |
| `DOC_TYPE` | `DOC_TYPE` | 오더 유형 |
| `SALES_ORG` | `SALES_ORG` | 영업 조직 |
| `DISTR_CHAN` | `DISTR_CHAN` | 유통 경로 |
| `DIVISION` | `DIVISION` | 제품군 |
| `SOLD_TO` | `SOLD_TO` | 고객 코드 |
| `NET_VALUE` | `NET_VALUE` | 오더 금액 |
| `CURRENCY` | `CURRENCY` | 통화 |
| `VBTYP_N` | `VBTYP_N` | 문서 유형 |

```
[📸 스크린샷: Mapping 탭 — 필드 선택 완료]
```

---

## 4-6. Task 2 추가 — 2LIS_11_VAITM (오더 아이템)

1. **Add Task** 버튼 클릭
2. Source Object: `2LIS_11_VAITM` 선택
3. Target Table: **`LT_OO_VAITM`** 입력
4. Load Type: `Initial and Delta`

필드 선택:

| Source 필드 | 설명 |
|------------|------|
| `DOC_NUMBER` | 판매오더 번호 (Key) |
| `S_ORD_ITEM` | 오더 아이템 번호 (Key) |
| `MATERIAL` | 자재 번호 |
| `MATL_GROUP` | 자재 그룹 |
| `SHORT_TEXT` | 자재 설명 |
| `BASE_UOM` | 기본 단위 |
| `SALES_UOM` | 판매 단위 |
| `ORD_QTY` | 오더 수량 |
| `NET_VALUE` | 아이템 금액 |
| `REJ_REASON` | 거부 이유 |

---

## 4-7. Task 3 추가 — 0CUSTOMER_ATTR (고객 마스터)

1. **Add Task** 버튼 클릭
2. Source Object: `0CUSTOMER_ATTR` 선택
3. Target Table: **`LT_OO_CUST`** 입력
4. Load Type: `Initial and Delta`

필드 선택:

| Source 필드 | 설명 |
|------------|------|
| `CUSTOMER` | 고객 코드 (Key) |
| `NAME` | 고객명 |
| `COUNTRY` | 국가 |
| `REGION` | 지역 |

```
[📸 스크린샷: Task 3개 추가 완료 — RF 전체 구성 화면]
```

---

## 4-8. RF 저장 및 Deploy

1. **Save** 버튼 클릭
2. **Deploy** 버튼 클릭 → 상태: `Deployed` 확인

```
[📸 스크린샷: Deploy 완료 — 상태 Deployed]
```

---

## 4-9. Initial Load 실행

1. **Run** 버튼 클릭 → `Initial Load` 선택
2. 상태가 `Running` → `Completed` 로 변경될 때까지 대기 (2~5분)

```
[📸 스크린샷: Run 실행 후 Completed 상태]
```

3. **Data Preview**: `LT_OO_VAHDR` 클릭 → 데이터 확인

```
[📸 스크린샷: LT_OO_VAHDR 데이터 프리뷰]
```

> 💬 **강사 설명**:  
> "ERP에서 ABAP을 실행하지 않아도 데이터가 DSP로 들어왔습니다.  
> VBAK의 데이터가 `DOC_NUMBER`, `AUDAT` 등 Standard DataSource 필드명으로 들어온 것을 확인하세요."
