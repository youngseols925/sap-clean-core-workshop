# TO-BE 세션 ④ — Analytic Model (AM_OO_LITE)
> **소요시간**: 90분  
> **목표**: Fact View 위에 Analytic Model을 만들어 당월/전월/YTD 비교 분석을 완성한다

---

## 7-1. Analytic Model이란?

```
V_OO_LITE_F (Fact View)
      ↓
AM_OO_LITE (Analytic Model)
      ↑
  사용자 질의: "2024년 8월 기준으로 당월/전월/올해누계/작년누계 금액을 보여줘"
               → P_MONTH = '202408' 입력 한 번으로 자동 계산!
```

> 💬 **강사 설명**:  
> AM은 DSP의 핵심 분석 레이어입니다.  
> ABAP에서는 Aging 구간을 직접 코딩했지만,  
> AM에서는 "기준월 하나"만 입력하면 당월/전월/YTD가 **자동으로** 계산됩니다.

---

## 7-2. 설계 전 미리 보기 — 만들 것

```
AM_OO_LITE
  ┌─────────────────────────────────────────────────┐
  │ 차원 (Attributes)                                │
  │   VKORG, SPART, KUNNR, MATNR, MATKL,           │
  │   WAERK, AGING_GRP, AUDAT_YM                   │
  ├─────────────────────────────────────────────────┤
  │ 지표 (Measures) — BASE                          │
  │   ORD_AMT (오더 금액)                            │
  │   ORD_QTY (오더 수량)                            │
  ├─────────────────────────────────────────────────┤
  │ 지표 (Measures) — RESTRICTION                   │
  │   CURR_MONTH_AMT  (당월 금액)                    │
  │   PREV_MONTH_AMT  (전월 금액)                    │
  │   CURR_YTD_AMT    (올해 누계)                    │
  │   PREV_YTD_AMT    (작년 누계)                    │
  ├─────────────────────────────────────────────────┤
  │ 변수 (Variables)                                 │
  │   P_MONTH ← 사용자 입력 (예: "202408")           │
  │   RV_CURR_MONTH    ← 자동 계산 (= 202408)       │
  │   RV_PREV_MONTH    ← 자동 계산 (= 202407)       │
  │   RV_CURR_YR_JAN   ← 자동 계산 (= 202401)       │
  │   RV_PREV_YR_MONTH ← 자동 계산 (= 202308)       │
  │   RV_PREV_YR_JAN   ← 자동 계산 (= 202301)       │
  └─────────────────────────────────────────────────┘
```

## 7-10. RESTRICTION Measure 4개 추가

> **RESTRICTION = "특정 조건(기간)을 만족하는 행만 집계하는 지표"**  
>   
> SQL로 표현하면:  
> `SUM(CASE WHEN AUDAT_YM = '202408' THEN ORD_AMT ELSE 0 END)`

**Add Measure** → **Restricted Measure** 클릭:

### CURR_MONTH_AMT (당월 금액)

| 항목 | 값 |
|------|----|
| 이름 | `CURR_MONTH_AMT` |
| 레이블 | `당월 금액` |
| 기준 지표 | `ORD_AMT` |
| 필터 조건 | `AUDAT_YM = RV_CURR_MONTH` |

### PREV_MONTH_AMT (전월 금액)

| 항목 | 값 |
|------|----|
| 이름 | `PREV_MONTH_AMT` |
| 레이블 | `전월 금액` |
| 기준 지표 | `ORD_AMT` |
| 필터 조건 | `AUDAT_YM = RV_PREV_MONTH` |

### CURR_YTD_AMT (올해 누계)

| 항목 | 값 |
|------|----|
| 이름 | `CURR_YTD_AMT` |
| 레이블 | `올해 누계` |
| 기준 지표 | `ORD_AMT` |
| 필터 조건 | `AUDAT_YM >= RV_CURR_YR_JAN AND AUDAT_YM <= RV_CURR_MONTH` |

### PREV_YTD_AMT (작년 누계)

| 항목 | 값 |
|------|----|
| 이름 | `PREV_YTD_AMT` |
| 레이블 | `작년 누계` |
| 기준 지표 | `ORD_AMT` |
| 필터 조건 | `AUDAT_YM >= RV_PREV_YR_JAN AND AUDAT_YM <= RV_PREV_YR_MONTH` |


> 💬 **강사 설명**:  
> "P_MONTH를 `202408`으로 입력하면  
> 당월 금액 = 2024년 8월 오더만 합산  
> 전월 금액 = 2024년 7월 오더만 합산  
> 올해 누계 = 2024년 1월~8월 오더 합산  
> 작년 누계 = 2023년 1월~8월 오더 합산  
> 이게 자동으로 계산됩니다.  
>   
> ABAP에서 이걸 구현하려면 Selection Screen에 4개 기간 파라미터 + 4번 SELECT + ALV 구성이 필요합니다.  
> AM에서는 기준월 하나만 입력하면 됩니다."
